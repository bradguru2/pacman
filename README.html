Run the game

Fully extract the archive before starting the game. Do not run the executable from inside the ZIP or TAR file.

Windows: Double-click PacMan.exe.

Linux: Open a terminal in the extracted folder and run:

chmod +x PacMan
./PacMan

Controls

Hold the arrow keys to move Pac-Man through the maze.

.NET runtime

The Windows and Linux releases are self-contained .NET 8 applications. You do not need to install the .NET 8 runtime. A self-contained application still depends on native operating-system, graphics, windowing, and audio libraries.

Required package contents

Keep the Assets folder beside the PacMan executable. It contains audio required by the game.

On Windows, the preferred setup is to keep the required native DLLs beside PacMan.exe:

  • glfw3.dll provides the game window, OpenGL context, and keyboard input.
  • OpenAL32.dll provides audio playback.

Windows 10 or 11 (64-bit)

  1. Extract the entire Windows x64 release archive.
  2. Confirm that glfw3.dll is in the same folder as PacMan.exe. If it is missing, download the official GLFW 3.4 Windows 64-bit binaries. Extract lib-vc2022/glfw3.dll and place it beside PacMan.exe.
  3. Confirm that OpenAL32.dll is beside PacMan.exe. If it is missing, download the official OpenAL Soft 1.25.2 binary package. Extract bin/Win64/soft_oal.dll, rename it to OpenAL32.dll, and place it beside PacMan.exe.
  4. Run PacMan.exe.

If Windows reports that VCRUNTIME140.dll, MSVCP140.dll, or another Microsoft C/C++ runtime DLL is missing, install the official Microsoft Visual C++ x64 Redistributable. This is a system-wide installation and normally requires administrator permission.

The game also requires a working OpenGL graphics driver. Install the current driver supplied by Intel, AMD, NVIDIA, or the computer manufacturer if the window cannot create an OpenGL context.

Linux x64

The Linux release targets 64-bit, glibc-based distributions. It needs OpenAL, OpenGL, and either X11 or Wayland client libraries in addition to the native libraries required by .NET 8.

Ubuntu 24.04

sudo apt update
sudo apt install ca-certificates libc6 libgcc-s1 libgssapi-krb5-2 \
  libicu74 libssl3 libstdc++6 tzdata zlib1g \
  libopenal1 libglfw3 libgl1 \
  libx11-6 libxrandr2 libxinerama1 libxcursor1 libxi6 \
  libwayland-client0 libxkbcommon0

For Ubuntu 22.04, replace libicu74 with libicu70. For Debian 12, use libicu72. Package names can differ on other releases.

Fedora and related distributions

sudo dnf install ca-certificates glibc libgcc krb5-libs libicu \
  openssl-libs libstdc++ tzdata zlib openal-soft glfw mesa-libGL \
  libX11 libXrandr libXinerama libXcursor libXi \
  wayland-libs libxkbcommon

Use the graphics driver recommended by your distribution or GPU vendor. For additional distribution-specific .NET native requirements, see Install .NET on Linux. You do not need to install the .NET runtime itself.

Troubleshooting

  • Make sure the archive is fully extracted before starting the game.
  • Windows: GLFW cannot be loaded: put the 64-bit glfw3.dll beside PacMan.exe.
  • Windows: OpenAL cannot be loaded: put the 64-bit OpenAL32.dll beside PacMan.exe.
  • Linux: libglfw.so.3 cannot be found: install libglfw3 on Debian/Ubuntu or glfw on Fedora.
  • Linux: OpenAL cannot be found: install libopenal1 on Debian/Ubuntu or openal-soft on Fedora.
  • Linux: no display or OpenGL context: verify that an X11 or Wayland session is available and that the correct GPU driver is installed. Remote/headless shells normally cannot open the game window without display forwarding.
  • Linux: permissions error: run chmod +x PacMan.
